RubyRevaled

A shared diary of a pair programming experiment

RubyRevealed is a story about how two friends, Adam Dalton and Andy Davis, developed this blog. Ideally, if one were to follow this blog from start to finish at any time, you would have an exact copy of this site.

So, let's dive right in

DigitalOcean

It all starts with a Virtual Private Server (VPS). For this tutorial, we will be using the amazingly priced Digital Ocean to host our website. Create an account and then proceed by creating a new Droplet. We set our hostname to be "rubyrevealed" and selected the following option.

Okay, next you need to select the desired region, we simply choose New York 1 since it was closer to us. Lastly select a droplet image. For this tutorial, we'll be using Ubuntu 12.10 x64 Server, shown below.

Once you click "Create Droplet" at the bottom, you will have your root password emailed to you in plain text. So it is strongly advised that you change it. 

        $ ssh root@[ip-address]
        $ passwd
        Enter new UNIX password:
        Retype new UNIX password:
        passwd: password updated successfully

Okay, let's install Vim, update our apt package database, and install the latest updates (if there are any). 

        $ apt-get install vim-nox
        $ apt-get update
        $ apt-get upgrade

If you see that a new kernel gets installed as part of the updates, you should reboot the system afterwards: 

        $ reboot

Apache

Okay, time to install Apache. We will eventually be installing Ruby on Rails to build this blog, so we followed some of this Ubuntu community guide. 

        $ sudo apt-get install apache2 apache2-mpm-prefork apache2-prefork-dev

Okay, if you own your own domain name, this is where we directed "rubyrevealed.com" to point to our new DigitalOcean server. We purchased the domain through DreamHost.com. We found this How to Set Up a Host Name with Digital Ocean guide pretty useful. Step 2 shows the Digital Ocean nameservers that you'll want to set at the place you registered your domain (DreamHost for us). Execute Step 3 and then you'll most likely just need to wait a few hours for the new nameservers to kick in.
